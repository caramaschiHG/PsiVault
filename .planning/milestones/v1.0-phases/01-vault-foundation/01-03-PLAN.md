<objective>
Implement the audit/trust primitives that make PsiVault feel like a vault: structured audit events, visible session control, privacy-safe defaults, and calm re-auth for sensitive actions.

Purpose: later phases need reusable audit and privacy primitives, but the user also wants trust to be visible and understandable from the start. This plan creates those primitives and exposes the first trust-control surface.

Output:
- audit event contract and persistence helpers
- redacted logging/privacy-safe surface rules
- visible session list and revocation UI
- human-readable activity feed skeleton
- reusable sensitive-action re-auth guard
- architecture/runbook documentation proving the foundation is ready for later phases
</objective>
