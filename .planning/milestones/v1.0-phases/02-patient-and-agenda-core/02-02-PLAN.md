<objective>
Create the appointment domain: occurrence and recurrence modeling, lifecycle mutations, and the conflict rules that protect the agenda.

Purpose: Phase 2 scheduling must become trustworthy before the UI layer can consume it. This plan defines the appointment schema, occurrence and series contracts, overlap rules, status transitions, and recurrence edit scopes so later agenda views and quick next-session flows only consume stable domain behavior.

Output:
- appointment occurrence and weekly series schema
- explicit care-mode and status rules for appointments
- hard-block conflict engine shared by all scheduling mutations
- recurrence generation and scope-edit logic for weekly series
- audited create/reschedule/cancel/status-change mutation surface
- automated tests for conflicts, lifecycle changes, and recurrence scopes
</objective>
