  <behavior>
    Finance domain model behavior:
    - createSessionCharge(input, deps) creates a charge with status "pendente", amountInCents from input (or null), paymentMethod null, paidAt null
    - updateSessionCharge(charge, input, deps) updates status, amountInCents, paymentMethod; sets paidAt to deps.now when status transitions to "pago", clears paidAt when transitioning away from "pago"
    - deriveFinancialStatus([]) returns "no_data"
    - deriveFinancialStatus([{status:"pago"}]) returns "up_to_date"
    - deriveFinancialStatus([{status:"atrasado"}, {status:"pendente"}]) returns "overdue" (overdue takes priority)
    - deriveFinancialStatus([{status:"pendente"}]) returns "pending_payment"
    - deriveMonthlyFinancialSummary computes totalSessions (all charges), totalReceivedCents (sum of pago with non-null amount), totalPendingCents (sum of pendente+atrasado with non-null amount)
    - listByMonth uses UTC boundaries: from = Date.UTC(year, month-1, 1), to = Date.UTC(year, month, 1), filter: createdAt >= from && createdAt < to
    - findByAppointmentId returns null when no charge exists for that appointmentId

    Appointment model extension:
    - Appointment interface gains priceInCents: number | null, meetingLink: string | null, remoteIssueNote: string | null
    - createAppointment stores priceInCents and meetingLink from input (both default to null if not provided)
    - updateAppointment (or a new updateAppointmentOnlineCare function) can update meetingLink and remoteIssueNote
    - remoteIssueNote is valid on ONLINE appointments only (enforce via guard: throw if careMode !== ONLINE)

    Communication templates behavior (wa.me URL scheme):
    - buildReminderWhatsAppUrl: returns "https://wa.me/55{stripped phone}?text={encoded message}" with Portuguese template "Olá, {patientName}! Lembrando da sua consulta..."
    - buildRescheduleWhatsAppUrl: returns wa.me URL with Portuguese reschedule message including original date
    - buildDocumentDeliveryWhatsAppUrl: returns wa.me URL with Portuguese message referencing document type
    - When patientPhone is null/undefined, phone part in wa.me URL is empty string (still valid — WhatsApp prompts user to enter number)
    - buildReminderMailtoUrl: returns "mailto:?subject=...&body=..." with pt-BR copy
    - All phone formatting: strip all non-digits, prepend "55" country code

    Charge audit event:
    - createChargeAuditEvent with type "charge.created" produces metadata containing only chargeId and appointmentId (never amountInCents or paymentMethod — SECU-05)
    - createChargeAuditEvent with type "charge.updated" produces metadata containing only chargeId, appointmentId, and newStatus
  </behavior>
