<objective>
Establish the production database and ORM layer connected to Supabase for v1.1.

Purpose: This phase sets up the database connection pooling, Prisma schema mappings, and connection infrastructure necessary for all subsequent phases to migrate from in-memory repositories to persistent PostgreSQL storage.

Output:
- Dual connection strategy implemented (`DATABASE_URL` with pooler and `DIRECT_URL`).
- Prisma Client singleton pattern implemented to prevent connection leaks during development.
- Prisma schema refactored to use UUIDs, `@map` for snake_case columns, and `@@map` for plural snake_case table names.
- Soft delete pattern established.
</objective>

<task type="auto" tdd="false">
  <name>Task 2: Refactor Prisma Schema Conventions</name>
  <files>prisma/schema.prisma</files>
  <action>
Refactor all existing models in `prisma/schema.prisma` (e.g., Account, Workspace, Session, PracticeProfile, Patient, Appointment) to follow the new database conventions:
- Change `@default(cuid())` to `@default(uuid())` for all ID fields.
- Add `@map("snake_case_name")` to all fields currently using camelCase (e.g., `createdAt DateTime @map("created_at")`).
- Add `@@map("plural_snake_case_table_name")` to map models to Postgres-idiomatic table names (e.g., `@@map("patients")`).
- Ensure any model requiring logical deletion implements `deletedAt DateTime? @map("deleted_at")`.
  </action>

  </verify>
