<task type="auto">
  <name>Task 1: Criar smoke test de conexão</name>
  <files>tests/infra-connection.test.ts</files>
  <action>
Criar o arquivo tests/infra-connection.test.ts com um smoke test que verifica db.$connect() sem erros.

O teste deve usar it.skip por padrão para não quebrar CI sem credenciais reais. Quando credenciais estão disponíveis (.env preenchido), o desenvolvedor pode rodar com INFRA_TEST=true para ativar.

Conteúdo do arquivo:

```typescript
import { describe, it, expect } from "vitest";
import { db } from "@/lib/db";

// Smoke test — requires real Supabase credentials in .env
// Run with: INFRA_TEST=true pnpm test tests/infra-connection.test.ts
const runInfra = process.env.INFRA_TEST === "true";

describe("Prisma connection smoke test (INFRA-03)", () => {
  it.skipIf(!runInfra)("conecta ao banco via Supavisor sem erros", async () => {
    await expect(db.$connect()).resolves.not.toThrow();
    await db.$disconnect();
  });

  it.skipIf(!runInfra)("executa query básica sem connection exhaustion", async () => {
    const result = await db.$queryRaw`SELECT 1 as ok`;
    expect(result).toBeDefined();
    await db.$disconnect();
  });
});
```

Notas:
- Usar it.skipIf(!runInfra) do Vitest (não it.skip estático)
- Chamar db.$disconnect() depois de cada teste para liberar conexão
- Importar db via @/lib/db (alias @/ já configurado no vitest.config.ts)
