# Milestone v1.3: Performance

**Status:** ✅ SHIPPED 2026-04-22
**Phases:** 23–26
**Total Plans:** 8

## Overview

Eliminação da lentidão sistêmica do PsiVault — navegação com full-reload, caching desabilitado no vault, ~44 queries desnecessárias no /financeiro, N+1 na agenda e dados sensíveis trafegando em listagens.

## Phases

**Goal**: Usuário navega entre seções sem full page reload, e o app aproveita o caching padrão do Next.js em todas as páginas vault
**Depends on**: Nothing (first phase of milestone)
**Plans**: 2 plans

Plans:
- [x] 23-01-PLAN.md — Migrar vault-sidebar-nav.tsx e bottom-nav.tsx para Link (NAV-01)
- [x] 23-02-PLAN.md — Remover force-dynamic e envolver session/client com React.cache() (CACHE-01, CACHE-02)

**Details:**
- Sidebar e bottom-nav: `<a href>` → `<Link href>` (signout mantido como `<a>` — API route)

- `createClient()` e `resolveSession()` wrapped com `React.cache()`

## Milestone Summary

**Key Decisions:**

- `React.cache()` é request-scoped — não persiste entre requests, correto para dados de auth
- `/api/auth/signout` mantido como `<a>` — API route não deve usar `<Link>`
- `listByWorkspaceAndDateRange` cobre um range arbitrário — grouping feito em memória
- `revalidatePath` com tipo `"page"` — preserva cache do vault layout compartilhado
- `findByAppointmentIds` retorna `Set<string>` — agenda só precisa de presença, não conteúdo
- Backup route usa `listAllByWorkspace` — único caller que precisa de `importantObservations` em bulk

**Issues Resolved:**

- Full page reload em cada clique de navegação (sidebar/bottom-nav com `<a>`)
- Cache do Next.js desabilitado por `force-dynamic` no vault layout
- ~44 queries por load do /financeiro
- N+1 de clinical notes na agenda
- `importantObservations` trafegando em todas as listagens de pacientes

**Issues Deferred:**

- Streaming visual de charts com `<Suspense>` — v1.4
- Column selection para endpoints de search de pacientes — v1.4
- Relatórios financeiros (DRE, IRPF export) — v1.4

**Technical Debt Incurred:**

- `exportFinanceCSVAction` ainda usa `listByWorkspaceAndMonth` (single-month, intencional — documentado)
