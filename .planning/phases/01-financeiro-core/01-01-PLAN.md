# Phase 01 — Financeiro Core Melhorado

## Objetivo
Transformar o controle financeiro existente em uma ferramenta prática e útil para o dia a dia do psicólogo.

## Análise do Estado Atual

### O que já existe
- **Model `SessionCharge`** no Prisma com status, amount, paymentMethod, paidAt
- **Domain layer** em `src/lib/finance/` (model, repository, store)
- **Página `/financeiro`** com resumo mensal e trend de 6 meses
- **Seção financeira no paciente** com lista de cobranças e form de atualização
- **`deriveMonthlyFinancialSummary()`** para agregação
- **Audit events** para lifecycle de cobranças

### Problemas identificados
1. Status "atrasado" não é calculado automaticamente — depende de ação manual
2. Marcar como pago requer abrir formulário (deveria ser 1 clique)
3. Sem filtros na lista de cobranças
4. Sem visão de inadimplência clara
5. Sem exportação de relatórios
6. Preço padrão da sessão não é facilmente configurável por paciente

## Task Breakdown

### T1.1 — Status "atrasado" automático
**Descrição:** Sessões com data passada e sem pagamento devem aparecer como "atrasadas" automaticamente.
**Arquivos afetados:** `src/lib/finance/model.ts`, `src/lib/finance/repository.ts`
**Detalhes:**
- Adicionar função `autoMarkOverdue(sessionCharges[])` que verifica `appointment.date < today && status === 'pendente'`
- Executar antes de exibir lista ou resumo
- Não persistir no banco (é cálculo em tempo real), ou persistir se fizer sentido

### T1.2 — Quick Pay (1 clique para marcar pago)
**Descrição:** Botão "✓" ao lado de cada cobrança pendente que marca como pago instantaneamente.
**Arquivos afetados:** `src/app/(vault)/financeiro/page-client.tsx`, `src/lib/finance/model.ts`
**Detalhes:**
- Server action `markChargePaid(chargeId, paymentMethod?)` 
- Botão com ícone ✓ que muda estado com feedback visual
- Modal opcional para confirmar forma de pagamento (ou usa padrão)
- Undo imediato se errou

### T1.3 — Filtros úteis na lista
**Descrição:** Filtros por status, paciente e período.
**Arquivos afetados:** `src/app/(vault)/financeiro/page-client.tsx`
**Detalhes:**
- Toggle: Todos | Pendentes | Pagos | Atrasados
- Select de paciente (quando houver muitos)
- Período: este mês (default), mês anterior, custom
- Filtros são combinados (AND)

### T1.4 — Indicador de inadimplência
**Descrição:** Cards no topo mostrando números críticos.
**Arquivos afetados:** `src/app/(vault)/financeiro/page-client.tsx`, `src/lib/finance/model.ts`
**Detalhes:**
- Card 1: "R$ X pendentes" (soma de todos pendentes + atrasados)
- Card 2: "Y cobranças atrasadas" (quantidade)
- Card 3: "R$ Z recebidos este mês" (já confirmados)
- Cores: vermelho para atrasados, amarelo para pendentes, verde para recebidos

### T1.5 — Resumo por paciente
**Descrição:** Na lista de cobranças, agrupar por paciente com totais.
**Arquivos afetados:** `src/app/(vault)/financeiro/page-client.tsx`, `src/lib/finance/model.ts`
**Detalhes:**
- Função `groupByPatient(charges[])` com totais (pago, pendente, atrasado)
- Expansível: clicar no paciente expande as cobranças individuais
- Ordenar por maior valor pendente

### T1.6 — Exportar relatório CSV
**Descrição:** Botão "Exportar" que gera CSV simples com cobranças do período.
**Arquivos afetados:** `src/app/(vault)/financeiro/page-client.tsx`, novo arquivo `src/lib/finance/export.ts`
**Detalhes:**
- CSV com colunas: Data, Paciente, Valor, Status, Forma Pgto, Pago Em
- Filtros atuais são aplicados ao exportar
- Nome do arquivo: `financeiro-YYYY-MM.csv`
- Sem PDF (CSV é mais útil, abre no Excel)

### T1.7 — Preço padrão por paciente
**Descrição:** Permitir configurar preço da sessão por paciente (herda do PracticeProfile como default).
**Arquivos afetados:** `src/lib/patients/`, `src/app/(vault)/patients/[patientId]/components/`
**Detalhes:**
- Adicionar campo `sessionPriceInCents` no Patient (nullable, fallback para PracticeProfile.defaultSessionPriceInCents)
- UI nas configurações do paciente para alterar
- Ao criar sessão para paciente, usa o preço do paciente

### T1.8 — Ordenação e UX da lista
**Descrição:** Lista ordenada por data (mais recente primeiro) com cores por status.
**Arquivos afetados:** `src/app/(vault)/financeiro/page-client.tsx`
**Detalhes:**
- Badge colorido: verde (pago), amarelo (pendente), vermelho (atrasado)
- Data da sessão visível
- Valor formatado em BRL (R$ 150,00)
- Hover mostra detalhes adicionais

## Dependências entre Tasks

```
T1.1 (status automático) ──┐
                            ├── T1.4 (indicadores) ── T1.5 (resumo por paciente)
T1.2 (quick pay) ───────────┘
                            
T1.3 (filtros) ──┬── T1.6 (export CSV)
                  └── T1.5 (resumo por paciente)

T1.7 (preço paciente) ── (independente, mas ideal antes de T1.2)

T1.8 (UX lista) ── (independente, melhora tudo acima)
```

## Ordem Sugerida de Execução

1. **T1.1** — Status automático (base para tudo)
2. **T1.7** — Preço por paciente (base para cobranças corretas)
3. **T1.4** — Indicadores de inadimplência (impacto visual imediato)
4. **T1.2** — Quick Pay (maior ganho de UX)
5. **T1.3** — Filtros (necessário para lista ser útil)
6. **T1.8** — Ordenação e UX (polimento)
7. **T1.5** — Resumo por paciente (agrupamento)
8. **T1.6** — Export CSV (feature final)

- [ ] Marcar como pago leva no máximo 2 cliques
- [ ] Cobranças atrasadas são destacadas automaticamente
- [ ] Filtros funcionam e combinam corretamente
- [ ] Export CSV gera arquivo válido
- [ ] Preço padrão do paciente é usado ao criar sessão
- [ ] Nenhum teste existente quebrado
- [ ] TypeScript sem erros (`tsc --noEmit`)
