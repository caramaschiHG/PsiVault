    **A) Criar `src/app/(auth)/components/submit-button.tsx`:**

    Componente client com "use client" no topo. Usa `useFormStatus` de `react-dom`. Props: `{ label: string }`.
    - Durante pending: exibe spinner SVG (animateTransform), todos os inputs do form ficam `disabled` via o `fieldset` wrapping trick OU via o `disabled` no botão que bloqueia o form submit.
    - A solução mais limpa para desabilitar inputs sem tornar toda a página client: envolver o form body em um único componente `AuthForm` client que contém os inputs e o `<SubmitButton>`. Mas o CONTEXT.md define que `<SubmitButton>` é o componente extraído. Portanto, o SubmitButton apenas desabilita o próprio botão — a página usa `<fieldset disabled={pending}>` para os inputs, implementado dentro do form via um wrapper client separado.

    Na prática, criar `SubmitButton` que:
    1. Chama `useFormStatus()` para obter `{ pending }`
    2. Renderiza `<button type="submit" disabled={pending}>` com `aria-busy={pending}`
    3. Quando `pending=true`: exibe `<SpinnerSVG />` no lugar do `label`
    4. Quando `pending=false`: exibe `label`

    SpinnerSVG inline:
    ```tsx
    function SpinnerSVG() {
      return (
        <svg width="20" height="20" viewBox="0 0 24 24" fill="none"
          stroke="currentColor" strokeWidth="2.5" strokeLinecap="round" aria-hidden="true">
          <circle cx="12" cy="12" r="10" strokeOpacity="0.25" />
          <path d="M12 2 a10 10 0 0 1 10 10">
            <animateTransform attributeName="transform" type="rotate"
              from="0 12 12" to="360 12 12" dur="0.8s" repeatCount="indefinite" />
          </path>
        </svg>
      );
    }
    ```

    Estilos do botão: `borderRadius: "16px"`, `padding: "1rem 1.2rem"`, `background: "#9a3412"`, `color: "#fff"`, `fontWeight: 700`, `border: 0`, `cursor: pending ? "not-allowed" : "pointer"`, `opacity: pending ? 0.7 : 1`, `display: "flex"`, `alignItems: "center"`, `justifyContent: "center"`, `width: "100%"` — todos como `satisfies React.CSSProperties`.
