# UX Round 3: Design System Hardening — Score 6.2 → 9.6+

## Objetivo
Elevar o score de UX de 6.2 para 9.6+ através de hardening do design system, eliminação de inline styles, acessibilidade total e polish de micro-interações.

## Contexto
- 351 testes passando, build OK, 7 fases GSD já completas
- 44 itens de melhoria identificados (7 críticos, 13 altos, 16 médios, 8 baixos)
- Problema raiz: **inline styles bypassam tokens CSS**, causando inconsistência visual, acessibilidade fraca e impossibilidade de dark mode

## Estratégia: 5 Fases Atômicas

Cada fase é independente, testável, e entrega valor mensurável. Ordem baseada em:
1. **Dependency graph** — tokens primeiro, componentes depois
2. **Risk/reward** — críticos primeiro (acessibilidade, consistência)
3. **Blast radius** — foundation antes de polish
4. **Testabilidade** — cada fase verificável com build + tests

## Fase 14: Inline Styles → Token Migration (Score: 7.2 → 8.4)
**Blast radius:** 50+ arquivos, migração sistemática
**Risco:** Médio (cada arquivo é independente)

### Strategy: Migration por domínio, não por arquivo
Cada sub-fase migra TODAS as ocorrências de um tipo de token em TODOS os arquivos.

### Tasks
| # | Task | Scope | Critério de Aceite |
|---|------|-------|-------------------|
| 14.1 | **Cores inline → CSS variables** | TODOS os arquivos | Zero cores hex hardcodadas (exceto CSS tokens definitions). Usar grep para verificar. |
| 14.2 | **Border-radius inline → tokens** | TODOS os arquivos | Zero `borderRadius` com valores não-token. Usar `--radius-sm/md/lg/xl/pill`. |
| 14.3 | **Padding/margin inline → tokens** | TODOS os arquivos | Zero `padding`/`margin` com rem hardcoded. Usar `--space-*`. |
| 14.4 | **Font-size inline → tokens** | TODOS os arquivos | Zero `fontSize` hardcoded. Usar `--font-size-*`. |
| 14.5 | **Shadow inline → tokens** | TODOS os arquivos | Zero `boxShadow` hardcoded. Usar `--shadow-*`. |
| 14.6 | **Transition inline → tokens** | TODOS os arquivos | Zero `transition` hardcoded. Usar `--transition-*`. |

**Migration pattern por arquivo:**
```tsx
// ANTES
const style = { padding: "1rem 1.25rem", borderRadius: "16px", color: "#1c1917" }

// DEPOIS
const style = { padding: "var(--space-4) var(--space-5)", borderRadius: "var(--radius-lg)", color: "var(--color-text-1)" }
```

**Exit gate:** `grep -rn '#[0-9a-fA-F]\{3,8\}' src/` retorna zero ocorrências em TSX (exceto globals.css)

## Risk Mitigation

| Risco | Mitigação |
|-------|-----------|
| Migração quebra visual | Commit por arquivo, diff visual antes de commit |
| Token missing | Criar token antes de migrar, não hardcode fallback |
| Test falha | Reverter commit, investigar, re-aplicar |
| Performance degrade | Benchmark antes/depois por fase |
