  <files>package.json</files>
  <action>
    Adicionar o script `postinstall` no bloco `"scripts"` do `package.json`:

    ```json
    "postinstall": "prisma generate && prisma migrate deploy"
    ```

    Manter todos os scripts existentes sem alteração. Apenas adicionar a nova entrada `"postinstall"`.

    Por que `prisma generate && prisma migrate deploy`:
    - `prisma generate`: garante que o Prisma Client está gerado (necessário após `pnpm install` no Vercel)
    - `prisma migrate deploy`: aplica migrations pendentes em produção usando `DIRECT_URL` (conexão direta, não Supavisor pool)
    - O Vercel executa `postinstall` automaticamente após `pnpm install` no build

    Nota: `prisma migrate deploy` usa `DIRECT_URL` do schema.prisma (conexão direta), não `DATABASE_URL` (Supavisor). Isso já está configurado no schema.prisma via `directUrl = env("DIRECT_URL")`.

    Remover o script `"prisma:generate"` pois agora está coberto pelo `postinstall` (ou manter se quiser compatibilidade — decisão do executor).
  </action>

  </verify>
