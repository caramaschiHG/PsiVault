# Plan 05 — UI Components

## Objective

Construir todos os componentes visuais da section Despesas seguindo UI-SPEC à risca:
drawer, modais, lista plana e agrupada, drop zone de comprovante, stat cards, toggle
de view. Integrar via `?tab=despesas` no `page-client.tsx` sem regredir Phase 19.

### Task 20-05-01 — Componentes de listagem + stat cards (read-only)

**Read first:**
- `.planning/phases/20-m-dulo-de-gest-o-de-despesas/20-UI-SPEC.md` § "Aba Despesas", "ExpenseList", "ExpenseGroupedList", "Stat Cards"
- `src/components/ui/list.tsx`, `src/components/ui/stat-card.tsx`
- `src/lib/expenses/format.ts` (formatBRL)

**Action:**

Criar 4 componentes em `src/app/(vault)/financeiro/components/`:

1. `expense-stat-cards.tsx`: 3 cards ("Despesas no mês", "A vencer", "Vencidas") com valores em `formatBRL` e `style={{ fontVariantNumeric: "tabular-nums" }}`. Recebe `expenses: Expense[]` e calcula totais inline.

2. `expense-list.tsx`: Lista plana. Cada row mostra descrição, badge da categoria, valor, due date, ícone se `receiptStorageKey != null`. Click → callback `onSelect(expense)`. Empty state com copy literal de UI-SPEC.

3. `expense-grouped-list.tsx`: Mesmo layout, agrupado por mês. Header sticky por grupo: `format(startOfMonth(dueDate), "MMMM yyyy", { locale: ptBR })`.

4. `expense-view-toggle.tsx`: Switch entre "Plana" e "Agrupada". Recebe `view` e `onViewChange`.

Todos os estilos via inline `satisfies React.CSSProperties` + CSS vars. Zero Tailwind. Sem gradientes. Sombras só `var(--shadow-sm)`.
