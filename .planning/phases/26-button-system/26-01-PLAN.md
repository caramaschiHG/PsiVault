# Phase 26-01: Criar Componente `<Button>` Genérico e Migrar SubmitButton UI

## Goal
Criar o componente canônico `<Button>` com variantes e refatorar o SubmitButton UI para usá-lo internamente.

## Tasks

### Task 1: Criar `src/components/ui/button.tsx`
- Componente `<Button>` com props: `variant` ('primary' | 'secondary' | 'ghost' | 'danger'), `size` ('sm' | 'md'), `asChild`, `isLoading`, `className`, `children`
- Usa classes CSS existentes: `.btn-primary`, `.btn-secondary`, `.btn-ghost`, `.btn-danger`
- `isLoading` prop: mostra spinner + desabilita botão + muda label automaticamente
- Suporte a `asChild` para renderizar como `<a>` quando necessário (links com estilo de botão)
- `:disabled` e `:focus-visible` herdados do CSS global

### Task 2: Adicionar `:disabled` global em `globals.css`
```css
.btn-primary:disabled,
.btn-secondary:disabled,
.btn-ghost:disabled,
.btn-danger:disabled {
  opacity: 0.6;
  cursor: not-allowed;
}
```

### Task 3: Adicionar `.btn-primary:active` consistente
```css
.btn-primary:active {
  transform: scale(0.97);
  transition: transform 60ms ease;
}
```

### Task 4: Refatorar `SubmitButton` UI para usar `<Button>`
- Manter `useFormStatus()` internamente
- Usar `<Button isLoading>` ao invés de lógica inline
- Manter props `label`, `pendingLabel`, `variant`, `className`
- Remover `baseStyle` / `pendingStyle` inline — tudo via CSS classes

### Task 5: Testar
- `pnpm test` — garantir que nenhum teste quebra
- Verificar visualmente em 2+ formulários do vault
