# Phase 26-03: Migrar Todos os Inline Button Styles para `<Button>`

## Goal
Eliminar todos os `submitButtonStyle`, `primaryButtonStyle` e estilos inline de botão dos formulários do vault, migrando para o componente `<Button>`.

## Tasks

### Task 1: Audit — encontrar todos os inline button styles
Arquivos conhecidos:
- `src/app/(vault)/agenda/page.tsx` — submitButtonStyle (linha ~864)
- `src/app/(vault)/agenda/components/quick-create-popover.tsx` — submitButtonStyle (linha ~310)
- `src/app/(vault)/appointments/components/appointment-form.tsx` — primaryButtonStyle (linha ~359)
- `src/app/(vault)/patients/[patientId]/components/reminders-section.tsx` — submitButtonStyle (linha ~268)
- `src/app/(vault)/sessions/[appointmentId]/note/components/note-composer-form.tsx` — submitButtonStyle (linha ~410)
- `src/app/(vault)/patients/[patientId]/documents/[documentId]/edit/page.tsx` — submitButtonStyle (linha ~178)

### Task 2: Migrar cada arquivo
Para cada arquivo:
1. Remover objeto de estilo inline (`submitButtonStyle`, `primaryButtonStyle`, etc.)
2. Trocar `<button style={...}>` por `<Button variant="primary" ...>`
3. Se estiver usando `useTransition` + `disabled={isPending}`, migrar para `isLoading={isPending}` do Button
4. Manter `pendingLabel` contextual (ex: "Agendando...", "Cancelando...", "Salvando...")

### Task 3: Verificar botões com `useTransition` (não `useFormStatus`)
Componentes com `useTransition` manual:
- `agenda/components/appointment-quick-actions.tsx`
- `agenda/components/quick-create-popover.tsx`
- `export-section.tsx`
- `reminders-section.tsx` (paciente)
- `workspace-backup-button.tsx`

Estes usam `disabled={isPending}` + texto condicional. Migrar para `<Button isLoading={isPending} loadingLabel="...">`.

- `pnpm test` — todos os testes devem passar
- `pnpm build` — build sem erros
