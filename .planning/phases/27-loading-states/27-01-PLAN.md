# Phase 27-01: Criar Componente `<Skeleton>` e Adotar Shimmer como Padrão

## Goal
Criar um componente `<Skeleton>` reutilizável com efeito shimmer e defini-lo como o único padrão de loading do produto.

## Tasks

### Task 1: Criar `src/components/ui/skeleton.tsx`
```tsx
interface SkeletonProps {
  width?: string;
  height?: string;
  borderRadius?: string;
  className?: string;
  style?: React.CSSProperties;
}
```
- Usa `skeleton-shimmer` keyframe (gradiente deslizante)
- Props opcionais para dimensões
- Suporte a `className` para overrides
- Default: width 100%, height 1rem, borderRadius var(--radius-sm)

### Task 2: Definir shimmer CSS via classe em globals.css
Adicionar classe `.skeleton-shimmer` que aplica:
```css
.skeleton-shimmer {
  background: linear-gradient(90deg, #f0ebe2 25%, #e8e2d9 50%, #f0ebe2 75%);
  background-size: 200% 100%;
  animation: skeleton-shimmer 1.4s ease infinite;
  border-radius: var(--radius-sm);
}
```

### Task 3: Documentar decisão
- Shimmer > Pulse porque: mais refinado visualmente, comunica "carregando" de forma mais clara
- Pulse será removido após migração de todos os loading.tsx
