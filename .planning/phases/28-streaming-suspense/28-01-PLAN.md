Create 2 files:

1. **src/lib/react/use-streamed-promise.ts**: A thin, typed wrapper around React 19's `use` API.
```typescript
import { use } from "react";

export function useStreamedPromise<T>(promise: Promise<T>): T {
  return use(promise);
}
```
That's it — no try/catch wrapper because `use` already throws on rejection, which is caught by ErrorBoundary. The value is in the explicit naming and type safety. Document with a short comment: "Unwraps a Promise inside a Client Component render. Must be called inside a Suspense boundary. Throws on rejection — catch with ErrorBoundary."

2. **tests/lib/react/use-streamed-promise.test.ts**: Unit test using Vitest and @testing-library/react.
```typescript
import { describe, it, expect } from "vitest";
import { render, screen } from "@testing-library/react";
import { useStreamedPromise } from "@/lib/react/use-streamed-promise";

function TestComponent({ promise }: { promise: Promise<string> }) {
  const value = useStreamedPromise(promise);
  return <span>{value}</span>;
}

describe("useStreamedPromise", () => {
  it("resolves and renders value", async () => {
    render(<TestComponent promise={Promise.resolve("hello")} />);
    expect(await screen.findByText("hello")).toBeInTheDocument();
  });
});
```
Note: Since this test relies on React's `use` which requires Suspense, wrap the render in a Suspense boundary with a fallback in the test setup.
