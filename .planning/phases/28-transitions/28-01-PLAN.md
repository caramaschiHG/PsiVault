# Phase 28-01: Page Transitions

## Goal
Adicionar transições de fade entre páginas do vault para que a navegação não seja instantânea e brusca.

## Tasks

### Task 1: Escolher abordagem
Opção A: Next.js `template.tsx` (App Router)
- Criar `src/app/(vault)/template.tsx` com `{children}` envolto em div com CSS animation
- Vantagem: nativo do Next.js 15, zero dependências
- Desvantagem: animação só no mount, não no unmount

Opção B: CSS-based com `view-transition` API
- Usar `document.startViewTransition()` no client-side navigation
- Vantagem: mais fluido, captura unmount
- Desvantagem: suporte de browser variável

**Decisão:** Opção A (template.tsx) — mais simples, funciona em todos os browsers, consistente com o ethos do produto.

### Task 2: Criar `src/app/(vault)/template.tsx`
```tsx
'use client';

export default function VaultTemplate({ children }: { children: React.ReactNode }) {
  return (
    <div style={{ animation: 'fadeIn 200ms ease-out' }}>
      {children}
    </div>
  );
}
```

### Task 3: Adicionar keyframe `fadeIn` em globals.css
```css
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(4px); }
  to { opacity: 1; transform: translateY(0); }
}
```

### Task 4: Respeitar `prefers-reduced-motion`
Adicionar ao bloco `prefers-reduced-motion`:
```css
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; }
}
```
(já existe — apenas verificar que cobre o fadeIn)

### Task 5: Testar
- Navegar entre 3+ páginas do vault — verificar fluidez
- Verificar que loading states não conflitam com a transição
