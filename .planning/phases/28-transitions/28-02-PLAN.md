# Phase 28-02: Toast Slide-in, FAB Animation, Search Dropdown Fade

## Goal
Adicionar animações de entrada aos componentes que atualmente aparecem instantaneamente.

## Tasks

### Task 1: Toast slide-in
Arquivo: `src/components/ui/toast-provider.tsx`

Problema atual: toast aparece com opacity direto, sem movimento de entrada.

Solução:
- Adicionar `@keyframes toastSlideIn` em globals.css:
```css
@keyframes toastSlideIn {
  from { opacity: 0; transform: translateX(100%); }
  to { opacity: 1; transform: translateX(0); }
}
```
- Aplicar `animation: toastSlideIn 200ms ease-out` ao toast novo
- Manter `transition: opacity 0.5s ease` para o fade-out (já existe)
- Diferenciar: toasts novos têm `animation`, toasts fading têm `transition`

### Task 2: FAB entrance/exit
Arquivo: `src/app/(vault)/inicio/components/quick-action-fab.tsx`

Problema atual: FAB aparece/desaparece sem animação.

Solução:
- Adicionar `@keyframes fabEnter` em globals.css:
```css
@keyframes fabEnter {
  from { opacity: 0; transform: scale(0.8); }
  to { opacity: 1; transform: scale(1); }
}
```
- Aplicar `animation: fabEnter 200ms ease-out` ao FAB no mount
- Para exit: usar CSS transition de opacity 150ms no unmount (se possível com client component)

### Task 3: Search dropdown fade-in + slide-up
Arquivo: `src/app/(vault)/components/vault-sidebar-nav.tsx` (ou onde está a SearchBar)

Problema atual: dropdown de busca aparece instantaneamente.

Solução:
- Adicionar classe CSS `.search-dropdown` com:
```css
.search-dropdown {
  animation: searchDropdownFade 150ms ease-out;
}

@keyframes searchDropdownFade {
  from { opacity: 0; transform: translateY(8px); }
  to { opacity: 1; transform: translateY(0); }
}
```
- Se o dropdown é controlado por estado React, aplicar classe condicional
