# Phase 29-01: Server Actions — Financeiro

## Goal
Criar as server actions necessárias para as funcionalidades do finance page upgrade.

## Tasks

### Task 1: Action "marcar como pago"
- Criar `updateChargeStatusAction(chargeId: string, status: 'pago' | 'pendente' | 'atrasado')`
- Validar workspace do usuário
- Atualizar status no repository
- `revalidatePath('/financeiro')`
- Retornar `{ ok: true }` ou `{ error: string }`
- Registrar evento de auditoria (charge status changed)

### Task 2: Action "adicionar cobrança manual"
- Criar `createManualChargeAction(formData: FormData)`
- Campos: patientId, date, amountInCents, status (default: pendente)
- Validar workspace, patient exists, amount > 0
- Criar SessionCharge via finance repository
- `revalidatePath('/financeiro')`

### Task 3: Action "atualizar cobrança"
- Criar `updateChargeAction(chargeId: string, updates: { amountInCents?: number; date?: string })`
- Para edição de cobranças existentes
- Validar workspace

### Task 4: Server component para dados de trend
- Criar função `loadMonthlyTrends(workspaceId, last 6 months)`
- Usa financeRepo.listByWorkspaceAndMonth para cada mês
- Retorna array de `{ month, year, totalReceived, totalPending, sessionCount }`
- Pode ser chamada inline no page component do financeiro

### Task 5: Server action para exportação CSV
- Criar `exportFinanceCSVAction(year: number, month: number)`
- Busca todas as cobranças do mês
- Gera string CSV
- Retorna como string para download client-side
- Não pode ser fetch direto — precisa de blob download pattern
