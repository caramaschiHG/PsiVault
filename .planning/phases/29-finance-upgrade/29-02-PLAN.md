# Phase 29-02: UI — Ações, Formulário, Filtro

## Goal
Transformar a UI do finance page de read-only para interativa com ações reais.

## Tasks

### Task 1: Botões de ação em cada cobrança
Para cada cobrança na lista:
- Se status = "pendente" ou "atrasado": botão "Marcar como pago" (ghost button, verde)
- Se status = "pago": badge "Pago" sem ação (visual só)
- Ação: `useTransition` + `updateChargeStatusAction` + toast de confirmação
- Durante pending: botão mostra spinner (via `<Button isLoading>`)

### Task 2: Botão "Adicionar cobrança"
- Botão primary no header (ao lado do mês)
- Abre formulário inline expansível abaixo dos cards de resumo
- Formulário: dropdown paciente, input data, input valor (R$), botão Salvar/Cancelar
- Submit: `createManualChargeAction` + toast + revalidate

### Task 3: Filtro por paciente
- Dropdown acima da lista (após cards de resumo, antes da lista)
- Populado com todos os pacientes do workspace
- Opção "Todos os pacientes" como default
- Filtro client-side (não requer server round-trip)
- Estado via URL searchParams (`?patient=pat_xxx`) para bookmarkable

### Task 4: Agrupamento por paciente
- Quando há múltiplas cobranças do mesmo paciente no mês:
  - Agrupar sob header com nome do paciente
  - Lista de cobranças indentada abaixo
- Se filtro por paciente está ativo, mostrar só aquele paciente (sem agrupar)

### Task 5: Empty state com CTA
- Quando não há cobranças: EmptyState com botão "Adicionar primeira cobrança"
- Abre o formulário inline direto

### Task 6: Loading state
- Usar `<Skeleton>` consistente (Phase 27)
- Estrutura: 3 cards + filtro + 5 rows
