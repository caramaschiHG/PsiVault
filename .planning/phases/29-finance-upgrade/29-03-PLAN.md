# Phase 29-03: Exportação CSV + Gráfico de Tendência Mensal

## Goal
Adicionar exportação CSV e visualização de tendência mensal ao finance page.

## Tasks

### Task 1: Botão "Exportar CSV"
- Posição: header do financeiro, ao lado do título ou no month nav bar
- Ao clicar: `exportFinanceCSVAction` busca dados do mês atual
- Gera blob e trigger download client-side
- Formato CSV:
```
Paciente,Data,Status,Valor
"João Silva",08/04/2026,pago,R$ 150,00
"Maria Santos",05/04/2026,pendente,R$ 150,00
```
- Nome do arquivo: `psivault-financeiro-abril-2026.csv`

### Task 2: Gráfico de tendência mensal (CSS puro)
- Posição: abaixo dos cards de resumo, acima do filtro
- Visual: barras verticais simples (divs com height %)
- Dados: últimos 6 meses de receita total (received)
- Eixo X: labels abreviados (Nov, Dez, Jan, Fev, Mar, Abr)
- Eixo Y: valores implícitos (altura proporcional ao máximo)
- Tooltip no hover: mostra mês + valor exato
- Cor das barras: accent color do tema (var(--color-accent))
- Sem biblioteca externa — CSS puro com divs

### Task 3: Implementação do gráfico
```tsx
// Exemplo de estrutura
<div style={chartContainerStyle}>
  {months.map((m) => (
    <div key={`${m.year}-${m.month}`} style={barWrapperStyle}>
      <div 
        style={{ 
          ...barStyle, 
          height: `${(m.totalReceived / maxVal) * 100}%` 
        }}
        title={`${m.monthLabel}: ${currency.format(m.totalReceived / 100)}`}
      />
      <span style={barLabelStyle}>{m.monthAbbr}</span>
    </div>
  ))}
</div>
```

### Task 4: Responsividade do gráfico
- Desktop: barras lado a lado
- Mobile: manter horizontal mas reduzir largura das barras e espaçamento

### Task 5: Verificar consistência visual
- Gráfico deve parecer parte do design system (cores, bordas, espaçamento)
- Não usar gradientes decorativos (anti-padrão do produto)
- Manter simplicidade — é um gráfico informativo, não um dashboard analytics
