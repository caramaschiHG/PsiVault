    Ensure that `prefers-reduced-motion: reduce` comprehensively covers ALL animations in the app.

    **In `src/styles/motion.css`:**
    The existing `@media (prefers-reduced-motion: reduce)` block must cover:
    - `.motion-fade-in`
    - `.motion-fade-out`
    - `.motion-slide-up`
    - `.motion-slide-down`
    - `.motion-scale-in`
    - `.motion-stagger > *`

    Expand it to ALSO cover all app-specific animation classes that reference keyframes in motion.css. Add to the same media query block:
    ```css
    .vault-page-transition,
    .toast-enter,
    .fab-enter,
    .search-dropdown-enter,
    .side-panel-drawer,
    .notif-dropdown,
    .tab-panel,
    .kbd-overlay,
    .skeleton-shimmer {
      animation: none !important;
    }
    ```

    **In `src/app/globals.css`:**
    The existing `@media (prefers-reduced-motion: reduce)` block (around line ~1251) currently uses:
    ```css
    *, *::before, *::after {
      animation-duration: 0.01ms !important;
      animation-iteration-count: 1 !important;
      transition-duration: 0.01ms !important;
      scroll-behavior: auto !important;
    }
    ```

    This is good but add a comment above it explaining it catches everything else:
    ```css
    /* Catch-all: all animations and transitions become instant.
       Utility-specific overrides live in motion.css. */
    ```

    Also ensure the globals.css media query is at the END of the file (after all regular rules) so it has highest specificity for the catch-all.
