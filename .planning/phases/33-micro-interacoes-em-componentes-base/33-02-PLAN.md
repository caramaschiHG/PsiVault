<task type="auto">
  <name>Task 1: Update PasswordInput for floating label and error shake</name>
  <files>src/app/(auth)/components/password-input.tsx</files>
  <action>
    Read password-input.tsx fully.

    The floating label CSS uses the sibling selector `~` which requires the label to be a direct sibling of the `<input>`. The current `.auth-input-wrap` structure (a `<div>` containing both input and eye button) breaks this selector because the label would be outside the wrap.

    **A. Refactor DOM to place input, eye button, and label as siblings inside `.input-floating-label-wrap`.**

    Change the component to accept new props:
    ```typescript
    interface PasswordInputProps {
      name: string;
      label?: string;
      placeholder?: string;
      required?: boolean;
      errorShake?: boolean;
    }
    ```

    Render structure:
    ```tsx
    <div className="input-floating-label-wrap">
      <input
        className={`auth-input ${errorShake ? "input-error input-error-shake" : ""}`}
        type={show ? "text" : "password"}
        name={name}
        placeholder=" "
        required={required}
        style={{ paddingRight: "2.75rem" }}
      />
      <button
        type="button"
        className="auth-input-eye"
        onClick={() => setShow((s) => !s)}
        style={{ position: "absolute", right: "0.75rem", top: "50%", transform: "translateY(-50%)", margin: 0 }}
        aria-label={show ? "Ocultar senha" : "Mostrar senha"}
      >
        {/* existing eye icon */}
      </button>
      {label && <label htmlFor={name} className="input-floating-label">{label}</label>}
    </div>
    ```

    **B. Handle animation end to remove shake class.**
    Use `onAnimationEnd` on the input to remove the `.input-error-shake` class after the animation completes, preventing replay on re-render:
    ```tsx
    const [shake, setShake] = useState(errorShake);
    useEffect(() => { setShake(errorShake); }, [errorShake]);
    
    <input
      className={`auth-input ${shake ? "input-error input-error-shake" : ""}`}
      onAnimationEnd={(e) => {
        if (e.animationName === "inputShake") setShake(false);
      }}
      ...
    />
    ```
    Import `useEffect` and `useState` from "react".

    **C. Remove the old `.auth-input-wrap` container and its associated styles.**
    The eye button is now absolutely positioned within `.input-floating-label-wrap` (which has `position: relative`). Ensure no `.auth-input-wrap` wrapper remains.
  </action>

  </verify>
