## Motion Contract

All animations use CSS keyframes and tokens from `src/app/globals.css` and `src/styles/motion.css`. No JavaScript animation libraries.

### Tokens

| Token | Value | Usage |
|-------|-------|-------|
| `--duration-100` | 100ms | Button opacity transition, border color transition |
| `--duration-200` | 200ms | Toast enter animation, input shake |
| `--duration-300` | 300ms | Toast exit animation (max allowed) |
| `--ease-out` | ease-out | Enter animations (snappy) |
| `--ease-in-out` | ease-in-out | Shake animation |

### Toast Animation (FEED-01)

**Enter:** `toastSlideIn` keyframe — `translateX(100%) → translateX(0)` + `opacity: 0 → 1`
- Duration: `var(--duration-200)` (200ms)
- Easing: `var(--ease-out)`
- Class: `.toast-enter`

**Exit:** Fade only — `opacity: 1 → 0`
- Duration: `var(--duration-300)` (300ms)
- Easing: `var(--ease-out)`
- Implementation: React state `fading` + inline style `opacity: 0` with CSS transition
- DOM removal delayed by 300ms to allow animation completion

**Visibility duration:** ~3000ms before exit animation starts

**Stack behavior:** No animated repositioning. When a toast is removed, remaining toasts snap instantly to new position.

**Position:** Fixed, bottom-right, `bottom: 1.5rem`, `right: 1.5rem`
**Max simultaneous:** 4 toasts
**Z-index:** `var(--z-toast)` (500)

### Button Loading State (FEED-02)

**Visual state during `isLoading`:**
- Opacity: 0.7 (reduced from full)
- Spinner icon visible (14×14px SVG, `spin` keyframe)
- Optional `loadingLabel` replaces children text
- `aria-busy="true"` set automatically
- `disabled` attribute set automatically (prevents double-submit)

**Spinner animation:**
- Keyframe: `spin` (360deg rotation)
- Duration: 600ms linear infinite
- SVG: `strokeDasharray="22"`, `strokeDashoffset="8"` for arc segment effect

**Transition into loading:**
- Opacity transition: `var(--duration-100)` ease-out
- No layout shift — spinner is `flexShrink: 0` and inserted before text

**Standard pending labels (pt-BR):**
| Context | Label |
|---------|-------|
| Form submit (default) | "Salvando..." |
| Form submit (danger) | "Excluindo..." |
| Generic action | "Carregando..." |
| Search/filter | "Buscando..." |

### Skeleton Shimmer (FEED-03)

**Animation:** `skeleton-shimmer` keyframe — background-position shift
- Gradient: `linear-gradient(90deg, #f0ebe2 25%, #e8e2d9 50%, #f0ebe2 75%)`
- Background size: `200% 100%`
- Duration: **1.6s** (looping, organic pace — outside interaction token scale)
- Easing: `ease` (built-in, smooth sine-like)

**Visual contract:**
- Colors close to app background (`--color-bg: #f7f3ed`) — subtle, not high-contrast
- No border-radius override (inherits from Skeleton component prop)
- Delay support via `animationDelay` prop for staggered skeletons

### Spinner Component (FEED-04)

**Standalone spinner specs (extracted from Button):**
- Size variants: `sm` (14px), `md` (20px), `lg` (28px)
- Stroke width: 2px
- Color: `currentColor` (inherits from parent)
- Animation: `spin 0.6s linear infinite`
- Accessible: `aria-hidden="true"` when decorative, `role="status"` + `aria-label` when standalone

### Form Error Feedback (FEED-05)

**Visual feedback on error:**
1. **Border color transition:** `var(--color-error-border)` → `var(--color-error-text)`
   - Transition duration: `var(--duration-100)`
2. **Shake animation:** `inputShake` keyframe
   - Movement: `0 → -2px → +2px → 0`
   - Duration: `var(--duration-200)`
   - Easing: `var(--ease-in-out)`
   - Iteration: 1 (single pulse, does not repeat)
   - Class: `.input-error-shake`
3. **Combined trigger:** Both applied simultaneously when error state is detected

**Timing:** Immediate (no delay) upon validation failure.

## Component State Contracts

### Toast States

| State | Visual | Motion |
|-------|--------|--------|
| Entering | Full opacity, left border colored | `toastSlideIn` 200ms ease-out |
| Visible | Static, dismiss button hoverable | None |
| Exiting | Opacity fading to 0 | `opacity` transition 300ms ease-out |
| Dismissed | DOM removed | Instant snap of remaining stack |

### Button States

| State | Visual | Motion |
|-------|--------|--------|
| Idle | Full opacity, no spinner | None |
| Loading | Opacity 0.7, spinner + optional label | Opacity transition 100ms |
| Success (post-action) | Returns to idle, toast triggered | See Toast Entering |
| Error (post-action) | Returns to idle, form shake triggered | See Form Error |

### Input States

| State | Visual | Motion |
|-------|--------|--------|
| Valid | Default border | None |
| Focus | Border accent + glow | Border/shadow transition 100ms |
| Error | Border red + shake | Border color 100ms + shake 200ms |
| Error→Valid | Border returns to default | Border color 100ms (no shake) |

## Accessibility & Reduced Motion

### prefers-reduced-motion

All animations in this phase MUST degrade gracefully under `prefers-reduced-motion: reduce`:

| Animation | Reduced Motion Behavior |
|-----------|------------------------|
| Toast enter | Instant (no slide, opacity 1 immediately) |
| Toast exit | Instant DOM removal (no fade) |
| Button loading spinner | Hidden (static icon or none) |
| Skeleton shimmer | Static background color (`#f0ebe2`) |
| Input shake | No shake (border color transition only) |
| Spinner standalone | Hidden or replaced with text "Carregando..." |

**Implementation:** Covered by existing media query in `motion.css`:
```css
@media (prefers-reduced-motion: reduce) {
  .toast-enter, .skeleton-shimmer, .input-error-shake, [style*="animation: spin"] {
    animation: none !important;
    opacity: 1 !important;
    transform: none !important;
  }
}
```

### ARIA

| Element | Attribute | Value |
|---------|-----------|-------|
| Toast container | `aria-live` | `polite` |
| Toast container | `aria-atomic` | `false` |
| Toast item | `role` | `status` |
| Button loading | `aria-busy` | `true` |
| Spinner (standalone) | `role` | `status` |
| Spinner (standalone) | `aria-label` | `"Carregando..."` |
| Toast dismiss | `aria-label` | `"Fechar notificação"` |

## Anti-Patterns (Explicitly Prohibited)

- ❌ No Framer Motion, react-spring, or any JS animation library
- ❌ No full-screen loading spinners (skeletons are the standard)
- ❌ No animated toast stack repositioning (snap only)
- ❌ No toast enter animation >300ms
- ❌ No skeleton pulse (opacity blink) — must use gradient shimmer
- ❌ No repeating error shake (single pulse only)
- ❌ No spinner without `aria-busy` or `aria-hidden` consideration
- ❌ No loading state that disables cursor feedback
