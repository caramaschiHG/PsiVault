## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Server Action | FormData cruza de cliente não-confiável para Server Action |
| URL searchParams → Server Component | Query params são input não-confiável para page.tsx |
| Server → Database | Appointment lookup e document save cruzam boundary de persistência |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-01 | Tampering | createDocumentAction | mitigate | Validar `appointment.workspaceId === workspaceId` antes de usar appointmentId |
| T-40-02 | Tampering | createDocumentAction | mitigate | Validar `appointment.patientId === patientId` antes de vincular documento |
| T-40-03 | Tampering | createDocumentAction / page.tsx | mitigate | Validar `from` começa com `/` e não é protocol-relative (`//`) nem contém newlines |
| T-40-04 | Information Disclosure | new/page.tsx | mitigate | `findById(appointmentId, workspaceId)` garante que só appointments do workspace são visíveis |
| T-40-05 | Elevation | createDocumentAction | mitigate | `resolveSession()` já fornece workspaceId/accountId validados; reutilizar |
| T-40-06 | Repudiation | createDocumentAction | accept | Audit trail de document creation já existe (document.finalized); não precisa de audit extra para vínculo |
